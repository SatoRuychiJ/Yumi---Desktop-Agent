===== 1.认识模组目录各文件夹，子文件夹和文件 =====
[img] 我们可以看到，目录下面有：food，image，lang，pet，text文件夹和icon.png，info.lps文件。
让我们来分析一下他们的作用：
 food文件夹：食物文件夹。里面应包含一个或多个{name}.lps文件，它(们)负责掌管食物的属性，示例模组为了分类，创建了两份文件。您可以根据自己的需要，任意变更他们的数量（只要里面的数据正确），桌宠应该都能够正确读取食物属性。
 image文件夹：食物图标文件夹，其中，food.png是默认的食物图片，也就是说，当一个或多个food文件夹中{name}.lps中的食物没有定义图标时，food.png会挑起大梁，担当起它(们)图标的重任。而food子文件夹内应包含各食物的图标，他们应当与上文中{name}.lps所定义的食物相绑定。
https://steamcommunity.com/sharedfiles/filedetails/?id=3020923781  lang文件夹：语言和翻译文件夹, 文件夹名称对应的相应语言英文缩写。
https://steamcommunity.com/sharedfiles/filedetails/?id=3020922038 
 pet文件夹：桌宠动作文件夹，里面存放动物动作的各帧png图像
https://steamcommunity.com/sharedfiles/filedetails/?id=3021412497 
 text文件夹：预设对话文件夹。里面应包含的{name}.lps与food文件夹中的相类似，可以根据自身的情况创建一份或多份文件，不影响桌宠的读取。{name}.lps负责存储不同状态下桌宠的说话行为。https://steamcommunity.com/sharedfiles/filedetails/?id=3020939669 
 plugin 插件文件夹
===== 模组上传 =====
理论上，当您的模组文件夹位置正确(Steam\steamapps\common\VPet\mod\)，
且info.lps和icon.png文件配置正确时，您已经可以对模组进行上传操作了。


但是请注意：itemid和authorid最好不要加上，不要加，不要加，不要加！！ > _<   ！！！（它在模组上传的时候会自动补充）


右键桌宠---->系统---->设置面板---->MOD管理---->{您的模组}---->右下角操作---->上传至Steam
请确保您有良好的网络环境。
